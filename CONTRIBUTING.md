# Contributing to Awesome OpenClaw

Thank you for your interest in contributing! This list thrives on community curation.

## How to Contribute

### Adding a Skill

1. Ensure the skill is publicly available (GitHub repo or ClawHub)
2. Add it to the appropriate category in README.md
3. Include a brief description of what it does
4. Submit a PR with format: `Add skill: [skill-name]`

Example:
```markdown
- [skill-name](https://github.com/user/repo) â€” Brief description of what it does
```

### Adding a Tool

Tools are utilities that work WITH OpenClaw (not skills). Include:
- Name and link
- What problem it solves
- Installation/command if applicable

### Adding an Example Project

Have you built something with OpenClaw? We'd love to feature it!

Include:
- Project name and link
- One sentence description
- Tech stack (optional)
- Screenshot or demo link (optional)

### Adding a Tutorial

Link to blog posts, videos, or written guides about OpenClaw.

## Pull Request Process

1. Fork the repository
2. Create a new branch: `git checkout -b add-skill-name`
3. Make your changes
4. Ensure README.md renders correctly
5. Submit PR with clear description
